# 外骨骼硬件定义与技术规范

## 1. 系统核心硬件
### 1.1 主控单元
*   **平台**：Teensy 4.1 (Cortex-M7, 600MHz)
*   **控制频率**：100 Hz (10ms 周期)

### 1.2 电机系统 (FOC 伺服电机)
*   **通讯协议**：CAN 总线 V2.35
*   **波特率**：1 Mbps (标准帧)
*   **关节规格**：
    *   **髋关节 (ID 1)**：减速比 1:36，控制比例 1° = 3600 单位，控制 ID 0x141
    *   **踝关节 (ID 2)**：减速比 1:10，控制比例 1° = 1000 单位，控制 ID 0x142，采用连杆驱动

---

## 2. 关节定义与安全限位
### 2.1 踝关节 (Ankle)
*   **零点**：足部与小腿成 90° 中立位
*   **正方向 (Positive)**：背屈 (Dorsiflexion, DF)，向上抬脚
*   **负方向 (Negative)**：跖屈 (Plantarflexion, PF)，向下踩步
*   **安全限位**：-15° (PF) ~ +15° (DF)

### 2.2 髋关节 (Hip)
*   **零点**：垂直站立姿态
*   **正方向 (Positive)**：向前迈腿 (Flexion)
*   **负方向 (Negative)**：向后抬腿 (Extension)

### 2.3 步态相位（产品定义）
*   **产品主线为 4 相**（承重 / 支撑中期 / 蹬地推进 / 摆动）：是否允许踝部跖屈（A1）、背屈（A2）等助力，以 `getCurrentGaitPhase4()` 及非退化模式下的规则为准；调试与协议中的跖屈有效标志与 **PHASE4_PUSH_OFF** 对齐。
*   **2 相**（支撑 / 摆动）由底层检测器输出，作为 4 相检测器的输入与退化映射，**不单独作为**「是否 push-off」等产品策略的判据。
*   **退化模式**（`phase4` 检测不稳定时）：映射为粗粒度 4 相；固件默认**关闭**跖屈与背屈目标助力，避免在不可靠相位下施加大力矩。

## 3. CAN 通讯协议 (V2.35)
### 3.1 帧结构
*   **控制 ID**：`0x140 + MotorID`
*   **反馈 ID**：与控制 ID 相同，回复时间 < 0.25ms
*   **数据格式**：8 字节，小端序 (Little-Endian)

### 3.2 关键控制指令 (Write)
| 指令值 | 功能说明 | 数据格式 |
| :--- | :--- | :--- |
| **0x80** | 电机掉电 (Disable) | - |
| **0x81** | 电机停止 (Stop) | - |
| **0x88** | 电机使能 (Enable) | - |
| **0x9B** | 清除故障 (Clear Error) | - |
| **0xA3** | 位置控制 1 | 目标位置 (int32) |
| **0xA4** | 位置控制 2 (带速限) | 目标位置 (int32), 速度限制 (int16) |
| **0xA1** | 扭矩/力矩控制 | 目标电流 iq (int16) |

### 3.3 关键查询与反馈 (Read/Feedback)
| 指令值 | 回复内容 | 字段说明 |
| :--- | :--- | :--- |
| **0x92** | 多圈角度 | int64 (7 字节), 单位 0.01°/LSB |
| **0x9C** | 状态 2 | 温度 (int8), 电流 (int16), 速度 (int16), 编码器 (uint16) |
| **0x9A** | 状态 1 | 电压 (uint16), 电流 (uint16), 错误标志 (uint8) |

---

## 4. 调试与操作规范
1.  **启动流程**：清除故障 -> 电机使能 -> 设定控制模式 -> 下发指令。
2.  **安全保护**：
    *   系统实时监控电机反馈（温度、电流、速度）。
    *   发生异常（如卡滞、反向、超时）时，程序应自动启动软退出流程。
    *   硬件/通讯故障触发时，立即下发停止或掉电指令。
3.  **零点标定**：
    *   使用 `az` 命令进行踝关节零点标定。
    *   使用 `hz` 命令进行髋关节参考位初始化。
