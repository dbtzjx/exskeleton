# 康复外骨骼系统需求与设计文档 (V1.0)

## 1. 系统概述
本项目旨在开发一套面向中风恢复期患者的智能康复外骨骼控制系统。系统通过感应患者步态相位，在关键时机提供“最小必要”的助力，辅助患者纠正步态缺陷（如足下垂、蹬地无力），同时激发患者的主动参与意识。

### 设计哲学
*   **非抑制性辅助**：仅在患者能力不足时补充动力，不强制走预设轨迹。
*   **柔性与安全**：系统具备顺从性，遇到外部阻力时自动降载，避免二次伤害。
*   **数据驱动**：全过程数据记录与可视化，为康复进展评估提供依据。

---

## 2. 现有功能实现

### 2.1 下位机控制 (Firmware)
*   **硬件平台**：Teensy 4.1 (600MHz), CAN 总线 (1Mbps)。
*   **信号预处理**：
    *   髋/踝角度 EMA 滤波。
    *   实时差分速度计算与滤波。
*   **步态相位识别 (稳定)**：
    *   基于髋关节角度与速度的双阈值状态机。
    *   自适应步态周期计算 (`Ts`, `Tst`)，自动适应不同步速。
*   **踝关节辅助策略 (已完成)**：
    *   **摆动相背屈 (Swing DF)**：通过 S 曲线平滑引导踝关节到达目标背屈角，解决脚尖拖地问题。
    *   **蹬地辅助 (Push-off)**：在支撑相末期 (`75%-95%`) 触发大电流脉冲，辅助推进。
*   **安全管线 (Safety Pipeline)**：
    *   **斜率限制**：限制电流突变，确保动作丝滑。
    *   **异常检测**：实时监控速度反向、关节卡滞及通讯超时。
    *   **软退出 (Soft Exit)**：异常发生时，通过 `compliant` 模式平滑卸载力矩并进入冷却。

### 2.2 上位机交互 (PC GUI)
*   **实时可视化**：支持 17 种不同数据标志位的实时曲线绘制，支持选择性显示。
*   **数据记录**：支持以 JSON 格式录制完整步态数据，方便后期回放分析。
*   **步态特征识别**：自动识别并提取单一完整步态周期曲线。
*   **参数加载**：支持将 PC 端记录的标准步态轨迹下发至 Teensy 进行模拟测试。

## 3. 未来规划功能

### 3.1 髋关节助力整合 (下一阶段)
*   **目标**：在摆动相早期提供屈髋助力，减小患者抬腿负担。
*   **方案**：复用现有的摆动进度 `s`，在 `s < 0.4` 时段内按正弦窗函数提供比例电流助力。

### 3.2 参数持久化存储
*   **需求**：将角度零点 (`az`/`hz`)、控制阈值、助力强度等参数保存至 EEPROM。
*   **目标**：实现上电即用，无需每次重新标定。

### 3.3 交互性能优化
*   **目标**：进一步优化上位机大数据量下的绘图流畅度。
*   **方案**：采用更高效的数据降采样算法或多线程渲染。

## 4. 关键技术参数

| 参数项 | 说明 | 初始配置/取值 |
| :--- | :--- | :--- |
| 控制频率 | 主循环运行频率 | 100 Hz (10ms) |
| 踝背屈窗口 | 摆动相目标角度 | +2° ~ +8° |
| 蹬地触发窗口 | 支撑相进度区间 | 0.75 ~ 0.95 |
| 顺从电流阈值 | 异常检测触发电流 | 500mA (I1), 1000mA (I2) |
| 安全限位 | 踝关节逻辑角范围 | -15° ~ +15° |
| CAN 电流单位 | 协议 iq 对应关系 | 取决于驱动器 (示例: 15u = 1500mA) |
